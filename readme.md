# Team Finder

Платформа для поиска команды и проектов. Бэкенд-приложение на Django. Вариант 2 – навыки пользователей, фильтрация по навыкам.

## III. Инструкция по работе

### 1. Клонирование репозитория
```bash
git clone https://github.com/kharchenkois2004-bot/team-finder-ad
cd team-finder
```

### 2. Настройка виртуального окружения
```bash
python3 -m venv venv
```
После этого появится папка `venv`, где будут храниться зависимости проекта.

### 3. Активация окружения:

**Windows (PowerShell):**
```bash
venv\Scripts\Activate.ps1
```
**Windows (cmd):**
```bash
venv\Scripts\activate
```
**Linux/Mac:**   
```bash
source venv/bin/activate
```

### 4. Установка зависимостей из текстового файла `requirements.txt`:
```bash
pip install -r requirements.txt
```
После установки в окружении будут доступны все нужные библиотеки Django-проекта.

### 5. Создание `.env`

Файл `.env` содержит конфиденциальные настройки проекта — ключ Django, параметры БД и другие переменные. В файле '.env_example' указаны значения параметров по умолчанию. Можно их оставить без изменения, а можно настроить для собственных нужд.
**Windows (PowerShell):**
```bash
copy .env_example .env
```
**Linux/Mac:**  
```bash
cp .env_example .env
```

### 6. Запуск контейнера

Перед этим необходимо установить docker engine и docker compose, либо воспользоваться Docker Desktop. После этого нелбходимо выполнить команду:
```bash
docker compose up -d
```

### 7. Инициализация тестовых данных

Для заполнения тестовыми данными используются фикстуры. Для этого нужно загрузить файл test_data.json.
```bash
python manage.py loaddata test_data.json
```

### 8. Запуск сервера

По умолчанию сервер работает по адресу http://127.0.0.1:8000. Запуск проводится следующей командой:
```bash
python manage.py runserver
```

### 9. Авторизация на сайте

Для авторизации можно создать новый аккаунт, либо воспользоваться уже созданными. Email и пароли находятся в файлее test_users.txt. Также возможно использование администраторской панели Django.
